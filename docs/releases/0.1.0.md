# NoHardText 0.1.0

Stable first public release of NoHardText.

NoHardText is a CLI static analysis tool that detects hardcoded user-facing strings in React/TSX projects before they reach production.

## Highlights

- Free MIT-licensed CLI scan for React/TSX projects.
- Detects hardcoded JSX text and common user-facing attributes.
- Emits structured rule IDs for every finding.
- Supports JSON reports for CI and external tooling.
- Supports GitHub Actions annotations.
- Supports configurable severity thresholds through `--fail-on`.
- Supports configurable rules and custom component text props.
- Reports localization health score, grade, top issues, and ship decision.
- Includes scan quality regression coverage for high-trust detection.
- Published on npm as `@nohardcoding/nohardtext`.

## Rules included

| Rule ID | Name | Category | Severity |
|---|---|---|---|
| `NHT1001` | JSX Text | localization | high |
| `NHT1002` | Placeholder Attribute | localization | high |
| `NHT1003` | Title Attribute | localization | high |
| `NHT1004` | ARIA Label | accessibility | high |
| `NHT1005` | Alt Attribute | accessibility | high |
| `NHT1006` | Component Text Prop | localization | high |

## Install

```bash
npm install -D @nohardcoding/nohardtext
```

## Run

```bash
npx nohardtext scan src
```

## Recommended rollout

For existing projects, start in observe mode:

```bash
npx nohardtext scan src --json --output nohardtext-report.json
```

Review findings, tune configuration, and only then enable blocking mode:

```bash
npx nohardtext scan src --fail-on high
```

## CI example

```bash
npx nohardtext scan src --github-annotations --fail-on high
```

## Notes for early adopters

This is the first stable release.

The recommended production path is to evaluate NoHardText on a real branch first, review false-positive rates, and then decide whether to enable blocking CI.

## Known limitations

- React/TSX only.
- No ESLint plugin yet.
- No VS Code extension yet.
- No auto-fix yet.
- Official reusable GitHub Action is planned.
